Scheduling vs Smart Rotation

Choose the right automation feature for your use case:

Feature Manual Scheduling Smart Rotation
Purpose Run specific attacks on a schedule Continuous UEBA baseline/attack simulation
Users Single user per schedule Multiple users (up to 30) rotating
TTPs You choose campaigns/tactics Auto-selected based on phase
Duration One-time or recurring Continuous (months)
Best For Scheduled penetration tests Exabeam/UEBA demo environments

Manual Scheduling

Create scheduled tasks that run attack simulations automatically using Windows Task Scheduler.

Creating a Schedule

  1. Navigate to Scheduler in the sidebar
  2. Scroll down to the Manual Schedules section
  3. Click New Schedule button
  4. Fill in the schedule form (see fields below)
  5. Click Create Schedule

Schedule Form Fields

Field Description Example
Schedule Name Descriptive name for the schedule Daily Recon Scan
Execution Target What to execute (APT campaign or tactic) APT29, Discovery tactic
Execute As User to impersonate (from user pool) CORP\jsmith
Schedule Type When to run (once, daily, weekly, monthly) Daily
Start Date/Time When the schedule begins 2024-01-15 09:00
Days of Week For weekly schedules only Mon, Wed, Fri
Run Cleanup Clean up artifacts after each technique Checked

Schedule Types

Once

Runs a single time at the specified date/time. Good for one-off tests.

Daily

Runs every day at the specified time. Good for continuous testing.

Weekly

Runs on selected days of the week. Good for business-hours testing.

Monthly

Runs once per month on the specified day. Good for compliance checks.

Managing Schedules

Each schedule in the list shows:

  • Status badge - Active (green) or Disabled (gray)
  • Next run time - When it will execute next
  • Last run time - When it last executed
  • Technique count - Number of TTPs in the schedule

Schedule Actions

Button Action
► Run Now Execute the schedule immediately
❚❚ Enable/Disable Toggle schedule on/off
✎ Edit Modify schedule settings
🗑 Delete Remove the schedule
Note: Schedules are managed through Windows Task Scheduler. They persist even if MAGNETO is not running and will execute at the scheduled time.

Smart Rotation for UEBA

Smart Rotation is designed for Exabeam UEBA demo environments where you need to establish baseline behavior before triggering attack alerts.

How It Works

Smart Rotation automates a continuous cycle of user behavior simulation:

BASELINE

14 days

Normal TTPs

3 TTPs/day

➔

ATTACK

10 days

Attack TTPs

Day 1: 10 TTPs

After: 3-4 TTPs

➔

COOLDOWN

6 days

No activity

Reset cycle

➔ Repeat

Key Concepts

  • Execution-Based Progression: Users advance phases based on TTPs executed, not just calendar days
  • Day 1 Burst: First day of attack phase runs 10 TTPs to guarantee UEBA alert
  • Campaign Rotation: Each cycle uses a different APT campaign (6 campaigns = 6 months variety)
  • Gradual Enrollment: Users are added in batches to create natural staggering

User Phases

Phase Duration TTPs Used Purpose
BASELINE 14+ days Discovery, Reconnaissance (12 TTPs) Establish normal user behavior in UEBA
ATTACK 10+ days Execution, Persistence, Credential Access, etc. (43 TTPs) Trigger UEBA anomaly alerts
COOLDOWN 6 days None Allow alerts to clear before next cycle
PENDING Varies None Waiting for enrollment date

Phase Progression Requirements

Execution-Based: Users only advance when they've completed enough TTPs:
  • Baseline → Attack: 14+ days AND 42+ baseline TTPs executed
  • Attack → Cooldown: 20+ attack TTPs executed
  • Cooldown → Next Cycle: 6 days elapsed, counters reset

Adding Users to Rotation

Before adding users to Smart Rotation, they must exist in your User Pool (see Adding Users Guide).

  1. Navigate to Scheduler in the sidebar
  2. Find the Smart Rotation section at the top
  3. Click Add Users button
  4. Select users from your user pool (checkboxes)
  5. Click Add to Rotation

User Card Display

Each user in rotation shows a card with:

Element Description
Phase Badge Current phase (Baseline, Attack, Cooldown, Pending)
Day Counter Day X of Y in current phase
Progress Bar Visual progress through phase
Campaign Current APT campaign (e.g., APT29)
TTPs Executed Count of TTPs run in current phase

Gradual Enrollment

When adding multiple users, they are enrolled in batches based on maxConcurrentUsers:

Example with maxConcurrentUsers = 4:
- Batch 1 (4 users): Enrolled today, start baseline immediately
- Batch 2 (4 users): Enrolled tomorrow
- Batch 3 (4 users): Enrolled day after tomorrow
- etc.

This prevents all users from hitting attack phase simultaneously.

Removing Users

  1. Click the X button on a user card
  2. Confirm removal
  3. User is removed from rotation but stays in user pool

Configuration Options

Click the Configure button in the Smart Rotation section to adjust settings.

Setting Default Description
Baseline Days 14 Minimum days in baseline phase
Attack Days 10 Minimum days in attack phase
Cooldown Days 6 Days of inactivity between cycles
Day 1 Burst TTPs 10 TTPs to run on first attack day
Daily TTPs 3 TTPs to run on subsequent days
Min Baseline TTPs 42 Required baseline TTPs before attack phase
Min Attack TTPs 20 Required attack TTPs before cooldown
Execution Time 09:00 Daily execution time
Max Concurrent Users 4 Users to process per day
Pause on Weekends Off Skip Saturday and Sunday

Enabling Smart Rotation

  1. Add users to rotation (see above)
  2. Configure settings as needed
  3. Click the Enable toggle button
  4. Confirm to create Windows Scheduled Task

Once enabled:

  • Status badge shows ACTIVE
  • Windows Task Scheduler runs daily at configured time
  • Users automatically progress through phases
  • Console shows execution logs when running

Manual Execution

Click Run Now to execute today's rotation plan immediately (useful for testing).

View Today's Plan

The dashboard shows today's execution plan:

  • Which users will be processed
  • What TTPs will be executed for each user
  • Expected duration
Important: Smart Rotation requires users with valid credentials. Session-based users (token auth) cannot be used for scheduled execution.

TTP Classification

Smart Rotation automatically classifies TTPs into baseline and attack categories:

Baseline TTPs (12 techniques)

Normal user activities that establish behavioral baseline:

  • Discovery: System info, network connections, file searches
  • Reconnaissance: User enumeration, domain info

Attack TTPs (43 techniques)

Malicious activities that trigger UEBA alerts:

  • Execution: PowerShell, command line, scripting
  • Persistence: Registry keys, scheduled tasks
  • Privilege Escalation: Token manipulation, UAC bypass
  • Credential Access: Credential dumping, keylogging
  • Lateral Movement: Remote services, pass-the-hash
  • Defense Evasion: Disabling security, obfuscation
  • Exfiltration: Data staging, transfer
  • Command and Control: Encoded commands, web protocols
  • Collection: Screen capture, clipboard
  • Impact: Data destruction, defacement
Note: TTP classification is stored in data/ttp-classification.json and can be customized.

Best Practices

For Manual Scheduling

  • Test first: Run schedules manually before enabling automation
  • Stagger times: Don't schedule multiple jobs at the same time
  • Monitor logs: Check execution history for failures
  • Use cleanup: Enable "Run cleanup" to avoid artifact buildup

For Smart Rotation

  • Start small: Begin with 4-8 users, then scale up
  • Use AD users: Create dedicated AD accounts for realistic simulation
  • Verify UEBA: Confirm baseline TTPs don't trigger alerts
  • Monitor Day 1: Watch the first attack day for alert generation
  • Adjust timing: Set execution time during business hours for realism

General Tips

  • Credential testing: Test all user credentials before scheduling
  • Admin privileges: Ensure MAGNETO runs with admin rights for Task Scheduler
  • Disk space: Monitor log file growth over time
  • Maintenance: Review and clean up old schedules periodically

Troubleshooting

Schedule not running

  • Check Windows Task Scheduler for errors
  • Verify MAGNETO has admin privileges
  • Ensure the scheduled time has passed
  • Check if schedule is enabled (not disabled)

User stuck in phase

  • Check TTP execution count meets minimum threshold
  • Verify user credentials are valid
  • Look for execution errors in console/logs

No UEBA alerts generated

  • Ensure baseline phase completed (42+ TTPs)
  • Verify attack TTPs are in UEBA's detection rules
  • Check UEBA is receiving Windows events
  • Confirm Day 1 burst executed (10 TTPs)

Windows Task Scheduler errors

  • Run MAGNETO as Administrator
  • Check Task Scheduler service is running
  • Verify no conflicting task names in "MAGNETO" folder
